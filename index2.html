JJU Digital Library Project Plan

Professional Evaluation: 10/10

Project Title: JJU Digital Library

Overview:

JJU Digital Library is an online platform intended to provide students and faculty of Jigjiga University access to a wide range of academic resources, including e-books, research papers, theses, and journals. The goal is to create an efficient, scalable, and user-friendly digital library system that enhances academic engagement and research capabilities.

Phases of SDLC

1. Requirement Gathering and Analysis

Goal: Understand what the users (students, faculty, librarians) need.

Tasks:

  • Conduct interviews with stakeholders
  • Create user personas (including accessibility needs)
  • Draft user stories
  • Define functional and non-functional requirements
  • Document API contract for future scalability

Tools:

Google Forms / Typeform (surveys)
Google Docs (documentation)
Miro / Whimsical (brainstorming & diagrams)

2. Planning

Goal: Define roadmap and resources for the project.

Tasks:

  • Break the project into sprints
  • Assign tasks (if working in a team)
  • Set deadlines and milestones
  • Create budget estimate

Tools:

Trello / Jira (project management)
Google Calendar (timeline mapping)
Excel/Sheets (budget tracking)

3. System Design

Goal: Blueprint the system architecture and user interface.

Tasks:

  • Create wireframes (UI layout)
  • Design database schema
  • Define system architecture
  • Plan for accessibility (WCAG 2.1 AA)

Tools:

Figma / Adobe XD (UI/UX design)
dbdiagram.io (database schema)
Draw.io (system architecture diagrams)
WAVE (accessibility evaluation)

Deliverables:

  • ER Diagram
  • UML Diagrams (Use Case, Activity, Sequence)
  • Wireframes for all major pages
  • Accessibility compliance report

4. Implementation (Coding)

Frontend Stack:

  • HTML5, CSS3, JavaScript ES6+
  • React.js (with Vite for faster build times)
  • Tailwind CSS (utility-first framework)
  • Axios for API communication

Backend Stack:

  • Node.js with Express.js
  • RESTful API architecture
  • JWT for authentication
  • Rate limiting for API protection

Database:

  • MongoDB (NoSQL, cloud-hosted via Atlas)
  • Mongoose ODM
  • Index optimization
  • Regular backups

Other Components:

  • Authentication: Firebase Auth
  • File Storage: AWS S3
  • Search: Elasticsearch integration
  • Logging: Winston/morgan

Development Tools:

  • VS Code with ESLint/Prettier
  • Git & GitHub (branch protection)
  • Postman (API testing)
  • Docker for containerization
  • MVC design pattern

5. Testing (Enhanced)

Goal: Ensure the platform works correctly, securely, and accessibly.

Types of Testing:

  • Unit testing (Jest)
  • Integration testing (Supertest)
  • End-to-end testing (Cypress)
  • Accessibility testing (axe-core, WAVE)
  • Penetration testing (OWASP ZAP)
  • Load testing (k6)
  • Cross-browser testing

Quality Gates:

  • 90% code coverage required
  • Zero critical accessibility violations
  • API response time < 500ms
  • 100% of WCAG 2.1 AA success criteria
  • No high/critical security vulnerabilities

Tools:

Jest (JavaScript testing)
Postman/Newman (API testing)
Cypress (end-to-end testing)
axe-core (accessibility)
OWASP ZAP (security scanning)

Accessibility Compliance

The platform will comply with WCAG 2.1 AA standards to ensure equal access for all users, including those with disabilities.

Key Requirements:

  • Perceivable: Text alternatives, captions, adaptable content
  • Operable: Keyboard accessible, enough time, no seizures
  • Understandable: Readable, predictable, input assistance
  • Robust: Compatible with assistive technologies

Implementation Checklist:

Feature Implementation
Screen Reader Compatibility ARIA labels, proper heading structure
Keyboard Navigation Focus indicators, logical tab order
Color Contrast Minimum 4.5:1 for normal text
Text Resizing 200% without loss of content
Form Accessibility Proper labels, error identification
